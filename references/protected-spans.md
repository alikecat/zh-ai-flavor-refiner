# protected-spans：保护区规则

在诊断与改写之前，先确定哪些内容不能动。保真优先于风格。

## 1. 强保护

默认不得改动：

- 数字：样本量、比例、金额、年份、实验结果、指标、编号。
- 时间：日期、时间段、阶段名、版本时间。
- 名称：人名、地名、机构名、政策名、项目名、产品名、模型名。
- 学术：术语、缩写、理论名、引用、参考文献、表图标题、公式、变量。
- 技术：代码、命令、路径、文件名、接口、版本号、错误日志、配置项。
- 法务/公文：条款编号、责任主体、法定表达、流程节点。
- 叙事：人物口吻、方言、自纠、停顿、故意重复、叙述视角。
- 检测报告：颜色、段落编号、平台名称、风险标签（作为定位信息，不改写报告本体）。
- 技术环境：安装命令、路径、环境变量、文件树、版本号（不做自然化改写）。
- Skill 元数据：触发词、规则编号、清单项目（优先保持可执行性）。

## 2. 中保护

可轻微整理，但不得改变含义：

- 段落顺序、论点和论据对应关系、因果关系、转折关系、条件关系。
- 情绪强度和立场方向、作者自称、称呼、语气边界。

## 3. 低保护

可优先处理：

- 空泛开场、段末升华、机械连接词、无信息量过渡句。
- 反复出现的万能动词、过度整齐的三点并列。
- "本文将""由此可见""具有重要意义"等不承担信息功能的表达。

## 4. 操作流程

1. 标记强保护内容。
2. 判断中保护内容是否承载逻辑。
3. 只对低保护内容先动刀。
4. 若必须改中保护内容，输出前回读是否改变含义。
5. 不主动重写强保护内容，除非用户明确要求纠错。

## 5. 回读问题

- 是否把"可能"改成了"必然"？"相关"改成了"因果"？"部分样本"改成了"全部样本"？
- 是否把作者观点改成了外部事实？是否把原文的不确定性删掉了？
- 是否把人称、责任主体或时间顺序改错了？
