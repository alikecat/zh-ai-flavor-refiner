# detector-feedback：检测反馈处理规则

当用户提供外部检测报告、颜色标记、风险分数或“高 AI 率”段落时，作为定位和迭代规则。

## 1. 基本定位

检测反馈是一种外部信号，用来帮助定位需要人工修订的段落。它不是事实标准，也不能替代文本质量判断。

处理检测反馈时，只记录：

- 哪些段落被标记。
- 标记强度或颜色。
- 检测器可能对应的问题类型。
- 本轮要修改的范围。
- 修改后是否仍保持事实、术语和语体。

不承诺检测通过，不承诺具体分数下降，不声称可以绕过某检测系统。

## 2. 检测反馈模式流程

```text
输入：原文 + 检测报告/高亮段落
  ↓
1. 解析标记范围
  - 只定位高风险句/段
  - 已低风险段默认不动
  ↓
2. 归因
  - 词汇模板？句壳？篇章同构？空泛结论？句长过匀？语体错配？
  ↓
3. 设定强度
  - light/minimal：小修
  - medium/standard：局部重写
  - aggressive：结构重排
  ↓
4. 修改
  - 优先处理结构和信息密度
  - 再处理词汇和连接词
  ↓
5. 回读
  - 信息是否变？术语是否漂？语体是否降级？
  ↓
6. 输出
  - 修改稿
  - 修改类型摘要
  - 仍建议人工核验的位置
```

## 3. 高风险标记的常见归因

| 标记表现 | 可能原因 | 优先处理 |
|---|---|---|
| 整段被标红/高风险 | 段落结构过于标准化，观点句 + 解释 + 总结 | 改段落功能，删总结句，增加具体材料 |
| 多句连续命中 | 句长、连接词、句壳过匀 | 打散同构句式，减少机械连接 |
| 结尾高风险 | 升华套话、意义拔高 | 改成限制、下一步、具体价值 |
| 开头高风险 | 宏大背景、本文将 | 直接进入问题、场景或事实 |
| 学术段高风险 | 无源归因、理论壳、贡献空泛 | 补来源、降调、回到证据 |
| 摘要高风险 | 全部是目的方法结果结论模板 | 保持摘要规范，但压缩套话，增加具体对象和结果 |

## 4. 迭代规则

- 每轮只改高风险部分，不进行无差别全文替换。
- 每轮保留一个修改记录：处理了哪些问题族。
- 若同一段多轮仍高风险，不继续盲改，应检查是否缺少事实、数据或具体材料。
- 如果检测目标要求牺牲准确性，停止并说明冲突点。

## 5. 检测器导向的可用操作

这些操作可作为客观方法记录：

- 改变段落开头方式：事实、问题、限制、案例、动作，而不是抽象概括。
- 改变段落功能：不是每段都“论点 + 解释 + 升华”。
- 降低万能词密度：赋能、助力、推动、体现、彰显。
- 减少句壳复用：不是 A 而是 B、核心在于、由此可见。
- 增加原文已有具体信息：数据、时间、对象、条件、失败点。
- 处理句长过匀：合并或拆分，但不为随机而随机。

## 6. 禁止操作

- 不虚构文献、实验、数据、案例、作者经历。
- 不把术语改成近义词导致概念漂移。
- 不为了制造随机性加入错别字、病句或口水话。
- 不承诺任何检测器结果。
- 不把“降低检测风险”说成“保证通过”。
