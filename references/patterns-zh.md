# patterns-zh：中文机械感模式库

用途：作为后续自建 `机械感诊断与编辑` skill 的 `references/patterns-zh.md` 雏形。  
定位：不是敏感词替换表，而是“问题族 + 判断条件 + 局部操作”的中文规则库。

---

## 0. 使用原则

1. 先判场景，再激活规则。朋友圈、论文、公文、报纸综述、技术 README、小说、访谈稿不能用同一把刀。
2. 先划保护区，再改语言。数字、引用、术语、人名、机构名、时间、变量、命令、路径、公式、表图标题默认不动。
3. 看密度，不看单点。一个“此外”不是 AI 味，连续三段“此外 + 因此 + 由此可见”才是问题。
4. 优先局部改写。只有当结构本身造成模板感时才整段改。
5. 不发明事实。所谓“人味”只能来自原文已有细节、作者已有判断、具体语境和真实限制。
6. 不跨语体降级。论文不能被改成朋友圈，公文不能被改成小红书，技术文档不能被改成广告文案。

## 1. 句壳类

| 编号 | 模式 | 高风险触发 | 判断条件 | 操作 |
|---|---|---|---|---|
| Z-SHELL-01 | 二元对比壳 | 不是 A，而是 B；并非 A，而是 B；不在于 A，而在于 B；不只是 A，更是 B；与其说 A，不如说 B | A 只是铺垫，B 才是实际信息；或全文多次用同类结构 | 删除 A，直接说 B；若 A/B 都重要，改成具体关系 |
| Z-SHELL-02 | 本质/核心声明壳 | 真正重要的是；真正决定 X 的是；本质上；核心在于；底层逻辑是 | 把普通判断拔高为“洞见”，但缺少证据或机制 | 直接写对象、条件、机制、后果 |
| Z-SHELL-03 | 先后序列壳 | 先 A，再 B；第一步/第二步；从 A 到 B；接下来我会 | 顺序只是仪式性，并不影响结果 | 保留真正操作顺序；删除教程式铺垫 |
| Z-SHELL-04 | 理论起笔壳 | 基于 XX 理论；依据 XX 框架；根据 XX 观点 | 多个分析段都从理论名开始 | 现象/材料先行，理论移到段中或段后 |
| Z-SHELL-05 | 段末印证壳 | 此案例印证了；由此可见；这提示我们；不难发现 | 反复“总结 + 升华 + 点题” | 删除总结句；必要时改成下一段承接 |
| Z-SHELL-06 | 抽象问题壳 | 核心问题是；主要挑战在于；关键矛盾体现在 | 抽象标签替代了具体问题 | 写具体障碍、失败点、约束或矛盾 |
| Z-SHELL-07 | 时代背景壳 | 在当今快速发展的时代；随着 XX 不断发展 | 可以套到任何主题 | 删除；若确有背景，写具体变化 |
| Z-SHELL-08 | 泛化意义壳 | 具有重要意义；意义深远；提供新思路；前景广阔 | 结尾没有可验证内容 | 改成具体价值、限制、下一步或可检验命题 |

## 2. 词汇与动词类

| 编号 | 模式 | 高风险触发 | 操作 |
|---|---|---|---|
| Z-WORD-01 | 万能动词 | 赋能、助力、推动、促进、彰显、凸显、体现、打造、护航 | 问“具体做了什么”；换成具体动作或删除 |
| Z-WORD-02 | 空泛评价词 | 深刻、重要、关键、显著、全面、系统、深入、充分 | 无证据支撑则降调；有指标则写指标 |
| Z-WORD-03 | 伪学术动词 | 揭示了、印证了、反映了、说明了、表明了 | 一段只保留必要的一个；尽量具体化 |
| Z-WORD-04 | 商业黑话 | 闭环、抓手、落地、沉淀、链路、收口、收窄 | 工作汇报中谨慎保留；生活/公开文本压回普通动作 |
| Z-WORD-05 | 过度限定 | 可能在一定程度上潜在地 | 一个判断只保留一个限定 |
| Z-WORD-06 | 机械连接词 | 首先、其次、再次、此外、同时、综上所述 | 按密度处理；连续命中时改 |
| Z-WORD-07 | 系动词回避 | 扮演角色、充当功能、作为载体、发挥作用 | 能用“是/有/用来”说清就不用绕 |
| Z-WORD-08 | 同义词轮换 | 系统/平台/工具/方案来回换名 | 学术和技术文本保持术语稳定 |

## 3. 篇章结构类

| 编号 | 模式 | 表现 | 操作 |
|---|---|---|---|
| Z-STRUCT-01 | 段落同构 | 每段都是观点句 + 解释 + 总结 | 改至少一个段落功能：事实开头、问题开头、案例开头 |
| Z-STRUCT-02 | 列表过密 | 短文大量三点/五点/七点 | 真并列才列表，否则合并成段 |
| Z-STRUCT-03 | 等长等重 | 三项每项长度、句法、情绪完全一致 | 按重要性展开，允许两项或四项 |
| Z-STRUCT-04 | 空开场 | 下面我们来；本文将从；先说结论 | 直接进入内容 |
| Z-STRUCT-05 | 空结尾 | 未来可期；值得期待；持续推进 | 用事实、限制、下一步、可验证命题收束 |
| Z-STRUCT-06 | 挑战展望模板 | 尽管面临挑战，但仍前景广阔 | 写具体挑战和具体应对 |
| Z-STRUCT-07 | 过度整洁 | 句长接近，段落重量均匀 | 调整节奏，但不发明事实 |
| Z-STRUCT-08 | 旁白解释 | 这说明了什么；这背后其实 | 不增加信息就删；需要解释就写机制 |

## 4. 语体漂移类

| 编号 | 模式 | 表现 | 操作 |
|---|---|---|---|
| Z-REG-01 | 学术降级 | 论文被改成聊天口吻 | 保持正式、克制、术语稳定 |
| Z-REG-02 | 工程师腔外溢 | 生活对话写成 debug/report | 保留事实，删姿态 |
| Z-REG-03 | 客服腔 | 亲亲、希望这能帮到你、如有需要 | 删除服务姿态，直说处理结果 |
| Z-REG-04 | 小红书 AI 腔 | 家人们谁懂、建议收藏、评论区告诉我 | 非平台文案则删除 CTA 和情绪模板 |
| Z-REG-05 | 翻译腔 | 长定语链、被动堆叠、英文骨架 | 缩短主语和动作，回到中文语序 |
| Z-REG-06 | 公文误杀 | 把规范表达改口语 | 公文/法律极保守，只改明显残留 |
| Z-REG-07 | 品牌广告误杀 | 把短句、留白、祈使误判为 AI | 先问发布场景；不按普通长文改 |
| Z-REG-08 | 访谈/特稿误杀 | 自纠、方言、人物声口被磨平 | 真人声口优先，必要时停手 |

## 5. 学术专用类

| 编号 | 模式 | 表现 | 操作 |
|---|---|---|---|
| Z-ACAD-01 | 理论名堆段首 | 依据/基于理论反复起笔 | 现象先行，理论后置 |
| Z-ACAD-02 | 无源归因 | 研究表明、专家认为但无出处 | 有来源则具体化；无来源则删权威壳或标待补 |
| Z-ACAD-03 | 泛化贡献 | 填补空白、丰富理论、提供新路径 | 只保留有章节或数据支撑的贡献 |
| Z-ACAD-04 | 过度客观化 | 该设计体现了、该方法反映了 | 写为什么这样选、解决哪个问题 |
| Z-ACAD-05 | 术语漂移 | 同一概念多名 | 固定术语，不做无意义同义替换 |
| Z-ACAD-06 | 连接词误杀 | 因此/然而/此外全删导致断裂 | 连接词按逻辑保留 |
| Z-ACAD-07 | 结果升华 | 数据说明重大意义 | 回到数值、效应方向、解释边界 |
| Z-ACAD-08 | 展望空泛 | 未来研究应进一步深入 | 写具体变量、样本、方法、验证路径 |

## 6. 输出前回读清单

1. 原文每个具体事实是否仍可追溯？
2. 终稿是否新增了原文没有的数字、事件、人物、引用或私人细节？
3. 术语、缩写、变量、表图名是否漂移？
4. 语体是否跨档降级？
5. 是否把真人声口、访谈感、方言、自纠磨掉？
6. 是否还有空开场、空结尾、无源归因、泛化意义？
7. 是否为了“像人”制造了新的口语化、错别字或浮夸表达？
8. 是否所有段落又被改成同一种新模板？

## 6. 误伤与豁免规则

以下内容命中模式时，默认先标记为“可能误伤”，不直接改写：

| 类型 | 说明 | 处理 |
|---|---|---|
| 规则库/词表 | 文本正在列举“不是 A 而是 B”“由此可见”等模式 | 不按正文风险计分；检查分类是否清楚 |
| 代码块/命令/配置 | 示例命令、JSON、YAML、脚本说明 | 默认不改 |
| Markdown 表格 | 表格中常用于列举触发词、术语和对照项 | 单独看表格用途，不参与普通句式判断 |
| 直接引语/案例原句 | 作者故意引用问题句 | 不改引用本体，可在引用后给建议 |
| 学术规范表达 | “因此”“此外”“可能”等在论文中承担逻辑功能 | 看密度和功能，不因单点命中改 |
| Skill/Prompt 文档 | 指令文本天然会反复出现“不要/默认/优先/如果” | 按可执行性审校，不按散文自然度审校 |

关键判断：触发词是在“被文本使用”，还是在“被文本讨论”。后者属于元文本，应降权处理。

## 7. 检测器导向策略的客观记录

以下是公开 skill 中常见的检测器导向方法。记录它们的目的，是为了在用户提供检测报告时理解其工作流，不代表对具体检测结果作保证。

| 编号 | 方法 | 典型操作 | 适用前提 | 风险 |
|---|---|---|---|---|
| Z-DET-01 | 只改高风险段 | 根据红色/紫色/疑似段落局部回改 | 用户提供报告或标记 | 全文上下文可能不一致 |
| Z-DET-02 | 结构优先 | 先改段落开头、结尾和同构结构 | 段落整体被标高风险 | 可能改变段落重心 |
| Z-DET-03 | 句法角色调整 | 不改术语，把术语从固定主语位置挪到宾语/话题位置 | 学术论文、技术说明 | 需人工回读防止含义变动 |
| Z-DET-04 | 连接词降密度 | 减少“因此/从而/进而/由此”等连续功能重叠 | 连接词过密 | 误删会破坏逻辑 |
| Z-DET-05 | 信息密度错落 | 合并低信息句，展开关键论证句 | 段落过于等重 | 不可虚构材料 |
| Z-DET-06 | 句长波动 | 合并或拆分过匀句子 | 句长 CV 过低 | 不为随机而随机 |
| Z-DET-07 | 元文本豁免 | 对规则表、示例、检测报告降权 | 输入是 skill/报告/词表 | 若正文夹在表外，需分区处理 |

使用顺序：报告范围 → 归因 → 局部改写 → 保真回读 → 记录改动类型。
